Initial release of the Ikihs syntax highlight engine.

### Highlights

- **Native performance.** The Rust engine compiles to a native binary (CLI) or a native Node addon (napi-rs). There is no WASM and no interpreter overhead.
- **Shiki-compatible API.** The npm package `ikihsjs` mirrors `createHighlighter`, `codeToTokensBase`, `codeToHtml`, `loadTheme`, `loadLanguage`, `getLoadedLanguages`, `getLoadedThemes`, and `getTheme`.
- **Theme compatibility.** The engine reads VS Code `theme.json` with last-match-wins selector resolution. It also has 6 compatibility rules for Syntect-specific scopes (for example, `support.macro` and `meta.property.object`).
- **Measured compatibility.** The engine has 96% byte-level color match against Shiki v4 across 9 fixtures (Python, JavaScript, Rust, edge cases). The fixture comparison harness gives per-fixture diagnostics.
- **Scope classification.** The engine classifies scopes into 12 Ikihs categories: Keyword, String, Comment, Function, Variable, Number, Operator, Type, Property, Parameter, Decorator, Invalid. The `BuiltinScopeMapper` has ~60 default mappings. The mapper hides raw TextMate scopes from the public API to make the engine swappable.
- **CLI with JSON output.** `ikihs highlight` reads source from a file or stdin. It applies a VS Code theme and outputs tokens with byte offsets, hex colors, raw scopes, and Ikihs categories.

### Crates

- **`ikihs-core`** (`0.1.0`) — Core types, `HighlightEngine` trait, `ScopeMapper`, `BuiltinScopeMapper`, `Scope`, `ScopeCategory`, `RgbaColor`, `Theme`, `TokenColor`, `TokenSettings`. It has 16 unit tests.
- **`ikihs-themes`** (`0.1.0`) — `VscodeThemeParser` for VS Code `theme.json`, `TmThemeParser` skeleton, `ThemeRegistry`. It preserves `theme.type` (light and dark).
- **`ikihs-engine-syntect`** (`0.1.0`) — `SyntectEngine` that implements `HighlightEngine`. It loads 75 bundled TextMate grammars. It converts themes with reversed rule order for last-match-wins. It classifies scope stacks and outputs uppercase hex. It has 11 unit tests, 10 fixture integration tests (96% guardrail), and 6 diagnose tests.
- **`ikihs-cli`** (`0.1.0`) — CLI binary. `ikihs highlight [source] --lang --theme`. It reads stdin or a file and outputs pretty-printed JSON.

### Packages

- **`ikihsjs`** (`0.1.0`) — Native Node.js addon through napi-rs. The API matches Shiki v4. It has platform binaries for `x86_64-unknown-linux-gnu`. It has 11 tests. TypeScript declarations are in `shiki-types.d.ts`.
- **`@ikihs/compare`** (`0.1.0`) — Internal comparison harness. The `compare` subcommand runs Ikihs and Shiki side by side with a byte-level diff. `gen-fixtures` regenerates `.shiki.json` references from Shiki v4. It has an HTML report generator.
